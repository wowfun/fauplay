# 001 Slide - 自动播放产品规范

## 1. 背景与问题

当前版本已支持侧边预览窗格与全屏预览，但仍需手动切换媒体项，`README` 中也标注“幻灯片播放”未实现。  
本规范目标是在预览场景新增自动切换（自动播放）能力，减少手动翻页成本，支持连续浏览。

## 2. 目标与范围

### 2.1 目标

- 在预览时提供自动播放开关。
- 支持可调速度，默认值统一。
- 保持现有预览与快捷键行为稳定，不引入后端依赖。

### 2.2 范围内

- 生效场景：侧边预览窗格 + 全屏预览。
- 生效对象：媒体文件（图片/视频）。
- 切换策略：图片按间隔，视频播完切换。
- 循环策略：播放到末尾后回到第一项继续播放。
- 预览遍历支持两种顺序模式：`顺序循环` 与 `随机播放`（互斥切换）。
- 随机播放采用“不重复打乱”：单轮内不重复，轮末自动重洗牌。

### 2.3 范围外（非目标）

- 缩略图缓存与加载策略由 `specs/003-thumbnail/spec.md` 统一定义，本规范不重复约束。
- 不做刷新后自动播放状态持久化。
- 不新增后端服务或本地数据库能力。

## 3. 用户可见能力

- 自动播放开关（启用/暂停）。
- 播放速度可调：`1-10` 秒，默认 `3` 秒。
- 图片按设定间隔切换。
- 视频播放结束后切换到下一项。
- 播放到末尾时循环播放。
- 可切换播放顺序模式：`顺序循环` / `随机播放`。
- 随机模式下支持历史回退（上一张）与随机队列前进（下一张）。
- 非自动播放状态下，手动“上一张/下一张”同样遵循当前遍历模式。
- `P` 快捷键切换自动播放启停。
- `T` 快捷键切换遍历模式（顺序/随机，预览打开时生效）。
- `[` / `]` 快捷键按当前遍历模式切换上一张/下一张（预览打开时生效）。
- 页面失焦或切到后台时自动暂停，回前台自动恢复。
- 自动播放过程中手动上一张/下一张后，继续自动播放并重置计时。
- 顺序模式“记住上次选择”仅限当前页面会话，不跨刷新保留。

## 4. 功能需求（FR）

- `FR-01` 系统必须支持在预览场景开启与关闭自动播放。
- `FR-02` 自动播放必须同时适用于侧边预览窗格与全屏预览。
- `FR-03` 自动播放速度必须可配置为 `1-10` 秒，默认 `3` 秒。
- `FR-04` 当当前项为图片时，系统必须按当前速度定时切换到下一项。
- `FR-05` 当当前项为视频时，系统必须在视频自然播放结束后切换到下一项。
- `FR-06` 自动播放切换到媒体列表末尾时，系统必须循环到第一项继续播放。
- `FR-07` 用户按下 `P`（非输入焦点）时，系统必须切换自动播放启停状态。
- `FR-08` 当页面不可见或窗口失焦时，系统必须暂停自动播放。
- `FR-09` 当页面恢复可见且窗口重新聚焦时，系统必须恢复自动播放（若用户未手动关闭）。
- `FR-10` 自动播放中用户手动切换上一张/下一张后，系统必须保持自动播放开启并从当前项重新计时。
- `FR-11` 自动播放仅作用于媒体文件（图片/视频），目录与不可预览文件不参与自动切换。
- `FR-12` 自动播放设置在当前会话内有效；刷新页面后按默认值初始化（`3` 秒、关闭状态）。
- `FR-13` 自动播放与键盘/点击切换期间，网格选中高亮必须稳定，无明显闪烁或整网格重绘抖动。
- `FR-14` 系统必须提供预览遍历顺序模式切换：`sequential` 与 `shuffle`。
- `FR-15` `shuffle` 模式下，同一轮随机序列中不得重复播放同一媒体项。
- `FR-16` `shuffle` 一轮结束后必须自动重建新随机序列并继续播放。
- `FR-17` `shuffle` 模式下手动“上一张”必须按随机历史回退；“下一张”按当前随机队列前进（无论自动播放启停）。
- `FR-18` 媒体列表变化（筛选/目录变化）后，系统必须立即基于新列表重建随机队列。
- `FR-19` 顺序模式选择仅在当前页面会话内记忆，刷新后恢复默认顺序模式（顺序循环）。
- `FR-20` 当预览打开时，系统必须支持 `T` / `[` / `]` 快捷键分别用于切换遍历模式、上一张和下一张；输入框聚焦时不触发。

## 5. 验收标准（AC）

### 5.1 图片场景

- `AC-IMG-01` 开启自动播放后，图片按设定间隔连续切换。
- `AC-IMG-02` 在自动播放过程中修改间隔，下一轮切换按新间隔生效。

### 5.2 视频场景

- `AC-VID-01` 自动播放到视频项后，不按图片计时器抢先切换。
- `AC-VID-02` 视频自然结束后自动切换到下一项。
- `AC-VID-03` 视频播放失败时不会卡死在当前项（进入下一项或按技术设计指定的容错路径）。

### 5.3 循环场景

- `AC-LOOP-01` 自动播放到最后一项后进入第一项并持续播放。

### 5.4 快捷键场景

- `AC-HOTKEY-01` 非输入焦点下按 `P` 可以启停自动播放。
- `AC-HOTKEY-02` 输入框聚焦状态下按 `P` 不触发自动播放切换。
- `AC-HOTKEY-03` 视频 `Space` 播放/暂停行为保持不变，不与自动播放快捷键冲突。
- `AC-HOTKEY-04` 预览打开时，按 `T` 可切换遍历模式；输入框聚焦时不触发。
- `AC-HOTKEY-05` 预览打开时，按 `[` / `]` 可按当前遍历模式切换上一张/下一张。

### 5.5 失焦场景

- `AC-VIS-01` 标签页不可见或窗口失焦时自动暂停。
- `AC-VIS-02` 返回前台后自动恢复（前提是用户未主动关闭自动播放）。

### 5.6 手动干预场景

- `AC-MANUAL-01` 自动播放中手动点击上一张/下一张后，自动播放持续开启。
- `AC-MANUAL-02` 手动切换后图片计时从当前项重新开始。

### 5.7 网格稳定性场景

- `AC-GRID-01` 连续自动播放时，左侧网格选中高亮可持续跟随且无明显闪烁。
- `AC-GRID-02` 键盘导航或鼠标点击切换选中项时，不出现整网格频繁重渲导致的闪烁。

### 5.8 随机场景

- `AC-RAND-01` 开启随机模式后，单轮播放内无重复项。
- `AC-RAND-02` 单轮结束后自动进入下一轮随机序列。
- `AC-RAND-03` 随机模式下手动“上一张”可回到最近历史项，手动“下一张”按随机队列前进。
- `AC-RAND-04` 顺序模式切换入口可在顺序/随机间切换。
- `AC-RAND-05` 筛选或目录变化后随机队列即时重建，并可继续播放。
- `AC-RAND-06` 刷新页面后顺序模式恢复默认值（顺序循环）。

## 6. 默认值与一致性约束

- 默认播放间隔：`3` 秒。
- 末尾行为：循环。
- 默认遍历模式：`顺序循环`。
- 快捷键：`P`。
- 遍历快捷键：`T`（切换模式）、`[` / `]`（上一张/下一张）。
- 设置持久化策略：仅会话内生效，不跨页面刷新。
- 顺序模式“记住上次选择”仅限当前页面会话，不写本地持久化存储。
- 文档术语统一使用：预览窗格、全屏预览、自动播放。

### 6.1 术语/状态定义

- `playbackOrder: 'sequential' | 'shuffle'`：预览遍历顺序模式。
- `shuffleQueue: string[]`：当前轮随机队列，元素为媒体 `path`。
- `shuffleHistory: string[]`：已播放历史队列，用于随机模式下的手动回退。

## 7. 稳定技术约束（长期保留）

- 自动播放状态必须在 `App.tsx` 统一管理，避免侧边预览与全屏预览状态分叉。
- 自动播放调度必须遵循：图片按定时切换，视频按 `ended` 事件切换，视频错误可触发容错跳转。
- 自动播放仅对媒体文件生效，目录与不可预览文件不得参与自动切换。
- 手动“上一张/下一张”与自动播放触发的“下一张”必须共用同一遍历策略实现，避免行为分叉。
- 快捷键约束长期固定：`P` 控制自动播放，`Space` 保持视频播放/暂停语义。
- 随机播放必须采用“当前轮队列 + 历史栈”模型，确保随机前进与历史回退行为可预测。
- 随机队列重建触发条件固定为“媒体集合变化”（筛选结果或目录变化）。
- 可见性策略长期固定：页面不可见/失焦暂停，回到前台自动恢复（不改变用户启停意图）。
- 网格选中态更新必须遵循“最小更新”原则，避免顶层状态驱动整网格重渲；实现与排查参考 `docs/troubleshooting.md` 的“网格区域点击或键盘导航时闪烁”。

## 8. 测试与验收场景

1. 顺序模式回归：保持现有循环行为不变。
2. 随机模式一轮不重复验证。
3. 随机轮次结束后重洗牌并继续播放。
4. 随机模式在自动播放关闭时手动前进/回退行为正确。
5. 顺序模式切换入口可在顺序/随机间切换验证。
6. 列表变化（筛选、目录切换）后的随机队列重建验证。
7. 刷新后顺序模式恢复默认值验证。
8. 与现有 `P`、`Space`、`Esc` 的交互不冲突验证。
9. `T`、`[`、`]` 快捷键仅在预览打开时生效，且输入框聚焦时不触发。

# 101 Thumbnail Pipeline 缩略图管线规范

## 1. 目的 (Purpose)

定义 Fauplay 的网格缩略图管线行为契约（Thumbnail Pipeline Contract），统一缩略图触发、优先级、并发、去重、缓存一致性、尺寸档位与失败降级语义；本专题定位为“本地文件浏览器”中的媒体缩略图子能力规范。

## 2. 关键术语 (Terminology)

- 缩略图管线（Thumbnail Pipeline）
- 缩略图尺寸档位（Thumbnail Size Preset）
- 缩略图加载状态（Thumbnail Load State）
- 缩略图任务语义（Thumbnail Job Semantics）
- 缓存一致性（Cache Validity）
- 渐进更新（Progressive Update）

术语值映射（仅用于命名参考）：

1. 尺寸档位映射：默认档 = `auto`，256 档 = `256`，512 档 = `512`。

## 3. 范围与非目标 (In Scope / Out of Scope)

范围内：

1. 网格缩略图触发策略与加载优先级（媒体文件范围）。
2. 缩略图并发上限、请求去重与任务取消语义。
3. 缩略图缓存命中、失效与跨尺寸隔离语义。
4. 默认/256/512 三档尺寸行为与切档体验。
5. 单项失败降级与可见反馈。

范围外（非目标）：

1. 预览面板与全屏预览的媒体播放行为（归属 `100-preview-playback`）。
2. 网关协议字段与后端服务改造（归属 `001/002` 与基础约束）。
3. 具体实现方案、函数级算法与代码路径约束。
4. 本专题执行计划与任务拆解（本轮仅创建稳定 `spec.md`）。

## 4. 用户可见行为契约 (User-visible Contract)

1. 文件卡片默认显示占位态，缩略图生成中显示加载态。
2. 缩略图成功后展示结果，失败时显示可识别回退态。
3. 可见区域缩略图应优先于不可见区域加载。
4. 快速滚动时，系统应优先保障滚动和选择交互流畅。
5. 用户可切换缩略图尺寸档位：默认档、256 档、512 档。
6. 切换尺寸档位后，网格交互持续可用，不应出现整页阻塞。
7. 页面刷新后允许重新生成缩略图，不要求跨刷新保留结果。

## 5. 跨组件共享语义定义 (Shared Semantics)

1. 缩略图尺寸档位
   - 档位：默认档 / 256 档 / 512 档
   - 默认档语义：程序决定的策略位，不绑定固定像素值
2. 缩略图加载状态
   - 至少覆盖：占位、加载中、可用、失败
3. 缩略图任务语义
   - 至少覆盖：目标文件、尺寸档位、优先级、去重键、取消语义
4. 缓存一致性语义
   - 至少覆盖：缓存命中条件、文件变化后失效、跨尺寸隔离
5. 管线阶段语义
   - 至少覆盖：触发、排队、执行、完成/失败、回退

共享原则：

1. 同一缩略图请求必须具有可复用的去重语义，禁止重复生成同一目标结果。
2. 不同尺寸档位结果必须隔离，禁止跨档位错误复用。
3. 可见区优先是全管线统一规则，不得被预取任务反向抢占。

## 6. 功能需求 (FR)

1. `FR-THP-01` 系统必须保证可见区任务优先于不可见区任务。
2. `FR-THP-02` 系统必须按“接近可见/进入可见”触发缩略图任务，不得全量抢跑。
3. `FR-THP-03` 系统必须限制缩略图并发上限并支持配置，避免阻塞核心交互。
4. `FR-THP-04` 系统必须对同一目标请求执行去重，避免重复生成。
5. `FR-THP-05` 系统必须保证不同尺寸档位结果隔离，不得混用。
6. `FR-THP-06` 系统必须定义缓存一致性规则，文件变化后不得长期展示陈旧结果。
7. `FR-THP-07` 系统必须支持默认档、256 档、512 档三种尺寸档位。
8. `FR-THP-08` 切档过程中系统必须保持网格可滚动、可选择、可打开。
9. `FR-THP-09` 切换到高尺寸档位时，系统必须支持渐进更新，不要求全量同步完成后再渲染。
10. `FR-THP-10` 单项缩略图失败不得阻塞其他文件项，且失败状态必须可见。
11. `FR-THP-11` 缩略图主链路必须满足纯 Web 约束，不引入后端服务与跨刷新持久化缓存作为前置条件。

## 7. 验收标准 (AC)

1. `AC-THP-01` 大目录快速滚动时，网格无明显卡顿与整体抖动。
2. `AC-THP-02` 可见区文件缩略图优先加载，不可见区任务不抢占关键资源。
3. `AC-THP-03` 尺寸切换过程中，网格交互持续可用。
4. `AC-THP-04` 512 档位允许分批完成，UI 无冻结。
5. `AC-THP-05` 跨尺寸切换后结果与当前档位一致，无错图复用。
6. `AC-THP-06` 文件发生变化后重新进入视图，缩略图可更新而非长期陈旧。
7. `AC-THP-07` 单项失败时出现可识别回退态，其他项继续加载。
8. `AC-THP-08` 目录切换或筛选变化后，缩略图任务与结果与当前集合一致。

## 8. 默认值与一致性约束 (Defaults & Consistency)

1. 默认档是策略档，不固定像素值。
2. 三档尺寸（默认/256/512）是当前稳定对外档位，新增档位需先更新规范。
3. 默认并发上限为 `8`，并发上限必须可配置。
4. 缩略图缓存以运行时有效为基线，刷新后允许失效。
5. 规范允许“规范领先实现”，但行为目标与验收口径必须保持稳定可追踪。

## 9. 失败与降级行为 (Failure & Degradation)

1. 单项失败时必须回退到可识别占位态，并允许用户继续浏览。
2. 缩略图任务异常不得阻塞网格滚动、选择、打开等核心交互。
3. 当资源压力较高时，允许降低后台预取强度，但不可破坏可见区优先规则。
4. 当高尺寸档位计算开销过高时，允许延迟完成非关键项，但不可造成前台交互冻结。

## 10. 测试与验收场景 (Test Scenarios)

1. 大目录 + 快速滚动下的流畅性回归。
2. 可见区优先与不可见区让位验证。
3. 默认档与 256/512 档切换中的交互连续性验证。
4. 512 档位分批更新下的稳定性验证。
5. 多文件跨尺寸切换的一致性验证。
6. 文件内容变化后的陈旧缩略图更新验证。
7. 单项失败降级与其他项连续加载验证。
8. 目录切换与筛选变化后的任务集合一致性验证。

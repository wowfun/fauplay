# 101 Thumbnail Pipeline 实施计划

## 1. 目的

本计划用于将 [`spec.md`](./spec.md) 的缩略图管线契约落地为可执行增量，基于当前代码现状分阶段推进，确保性能与一致性在迭代中可验证、可回退。

## 2. 当前实现对照

### 2.1 已具备能力

1. 三档尺寸已存在：默认档、256 档、512 档（工具栏可切换）。
2. 基础运行时内存缓存已存在（`Map`）。
3. 网格虚拟化已存在，缩略图加载具备“随卡片挂载触发”的近似懒加载行为。
4. 缩略图失败时可回退到图标占位，不阻塞整体网格渲染。

### 2.2 主要差距（相对 `101` FR）

1. 尚无统一任务队列与并发上限控制。
2. 尚无“可见区优先 + 近可见触发”显式调度语义（当前由组件挂载时机隐式实现）。
3. 去重仅覆盖已完成缓存命中，未覆盖“同键并发进行中请求”。
4. 缓存键未显式纳入文件变化维度，存在陈旧缩略图风险。
5. 失败状态是隐式回退，缺少可观测的失败态语义与诊断入口。

## 3. 增量规范 (Delta)

### Delta THP-D1：管线骨架与并发控制

状态：`completed`

目标：

1. 建立统一缩略图任务入口，避免卡片直接并行生成。
2. 引入全局并发上限（默认 `8`，允许配置）。

落地点：

1. 新增管线模块（建议：`src/lib/thumbnailPipeline.ts`）。
2. `FileGridCard` 改为通过管线请求缩略图，不直接调用底层生成流程。

验收：

1. 同时触发大量卡片时，实际并行生成数不超过并发上限。
2. 网格滚动、选择与打开交互无明显卡顿。

---

### Delta THP-D2：触发策略与优先级调度

状态：`completed`

目标：

1. 将“可见区优先”从隐式行为提升为显式调度规则。
2. 将触发时机定义为“进入可见/近可见”，非全量抢跑。

落地点：

1. 在 `FileGridViewport` 使用 `react-window` 可见区信息（`onItemsRendered`）输出可见与近可见索引区间。
2. 管线层按优先级调度：可见区 > 近可见区 > 其他预取。
3. 卡片仅发起请求，不决定调度优先级。

验收：

1. 可见区项明显先于不可见区完成。
2. 快速滚动时，旧不可见任务不会长期抢占可见区资源。

---

### Delta THP-D3：去重与缓存一致性

状态：`completed`

目标：

1. 完成“进行中请求去重 + 已完成结果缓存”双层复用。
2. 明确文件变化后缓存失效语义，避免长期陈旧。

落地点：

1. 管线层新增进行中请求表（`inFlight`），同键并发请求复用同一 Promise。
2. 缓存键升级为“文件标识 + 文件变化维度 + 尺寸档位”组合。
3. 统一定义尺寸隔离规则，禁止跨档位复用。

验收：

1. 同一文件同一档位并发请求仅生成一次。
2. 文件变化后重新加载可生成新结果，不长期显示旧缩略图。
3. 不同尺寸档位切换不出现错图复用。

---

### Delta THP-D4：失败降级与可观测性

状态：`completed`

目标：

1. 将失败降级从“静默回退”收敛为可见状态语义。
2. 保留主链路可用，避免单项失败影响整体交互。

落地点：

1. `FileGridCard` 增加显式缩略图状态：占位、加载中、可用、失败。
2. 失败态展示可识别回退样式（图标/文案/提示之一），不阻塞卡片操作。
3. 在开发日志中输出最小必要诊断信息（失败类型、文件路径摘要、尺寸档位）。

验收：

1. 单项失败可被用户识别。
2. 连续失败场景下网格仍可滚动、可选择、可打开。

## 4. 实施顺序

1. 先做 `THP-D1`（先稳住并发上限）。
2. 再做 `THP-D2`（建立可见区优先调度）。
3. 再做 `THP-D3`（去重与缓存一致性）。
4. 最后做 `THP-D4`（失败态与可观测性收口）。

说明：

1. 每个 Delta 完成后做一次大目录滚动回归。

## 5. 关键默认值与约束

1. 默认并发上限：`8`。
2. 并发上限必须可配置。
3. 默认档（`auto`）是策略位，不固定像素值。
4. 仍保持纯 Web 与运行时缓存约束，不引入跨刷新持久化缓存。
5. 规范可领先实现，但每个 Delta 完成后必须同步更新 `CHANGELOG` 主要变化。

## 6. 验收与回归清单

1. 大目录快速滚动：无明显卡顿和整体抖动。
2. 可见区优先：首屏与视口内项优先完成。
3. 尺寸切换：默认/256/512 切换期间交互可持续。
4. 512 档位：允许分批完成，不冻结 UI。
5. 跨尺寸一致性：无错图复用。
6. 文件变化后：缩略图可更新。
7. 单项失败：可见回退，其他项连续加载。
8. 目录切换/筛选变化：任务集合与当前结果一致。

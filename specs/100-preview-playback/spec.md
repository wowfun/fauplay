# 100 Preview Playback 预览播放规范

## 1. 目的 (Purpose)

定义 Fauplay 的预览播放行为契约（Preview Playback Contract），统一侧栏预览与全屏预览在自动播放、遍历模式与快捷键上的一致语义；本专题定位为“本地文件浏览器”中的媒体子能力规范。

## 2. 关键术语 (Terminology)

- 预览播放（Preview Playback）
- 自动播放（Auto-play）
- 遍历模式（Playback Order）
- 顺序遍历（Sequential）
- 随机遍历（Shuffle）
- 预览表现面（Preview Surface）
- 预览播放状态（Playback State）

术语值映射（仅用于命名参考）：

1. 遍历模式值映射：顺序模式 = `sequential`，随机模式 = `shuffle`。
2. 预览表现面值映射：侧栏预览 = `panel`，全屏预览 = `lightbox`。
3. 预览播放状态值映射：停止 = `stopped`，播放中 = `playing`，因可见性暂停 = `paused_by_visibility`。

## 3. 范围与非目标 (In Scope / Out of Scope)

范围内：

1. 自动播放启停、速度区间与默认值（仅媒体文件集合）。
2. 顺序/随机遍历与上一项/下一项行为。
3. 侧栏预览与全屏预览的一致行为约束。
4. 预览相关快捷键（`P`、`T`、`Space`、`[`、`]`、`J`、`L`、`R`、`Esc`）语义。
5. 页面可见性变化与播放暂停/恢复语义。
6. 预览播放错误反馈与降级约束。

范围外（非目标）：

1. 不定义 JSON-RPC 字段、MCP 报文与错误码格式（归属 `002-contracts`）。
2. 不定义网关和插件内部实现机制（归属 `001-architecture`）。
3. 不定义像素级视觉设计与品牌样式细节（交互基线归属 `003-ui-ux`）。
4. 不定义本专题执行计划与任务拆解（本轮仅创建稳定 `spec.md`）。

## 4. 用户可见行为契约 (User-visible Contract)

1. 用户可在预览打开时启停自动播放。
2. 自动播放同时作用于侧栏预览与全屏预览。
3. 图片按播放间隔切换，视频在自然播放结束后切换。
4. 播放到末尾后循环到首项继续播放。
5. 用户可在顺序与随机遍历模式间切换。
6. 随机模式单轮内不重复，轮末自动重建随机序列。
7. 手动上一项/下一项与自动播放使用同一遍历语义。
8. 快捷键语义统一：
   - `P`：切换自动播放
   - `T`：切换顺序/随机
   - `Space`：视频播放/暂停切换
   - `[`：上一项
   - `]`：下一项
   - `J`：视频快退（按“快进步长”配置生效，支持按住连续触发）
   - `L`：视频快进（按“快进步长”配置生效，支持按住连续触发）
   - `R`：视频倍速循环切换（`1 -> 3 -> 5 -> 0.5 -> 1`，长按不连续切换）
   - `Esc`：按 `003-ui-ux` 关闭优先级关闭预览
9. 输入控件聚焦时，非全局快捷键不触发预览控制。
10. 预览播放失败必须用户可见，不允许静默失败。
11. 当前预览为视频时，预览头部必须展示“快进步长”与“倍速”控制项；图片/非媒体预览不得展示该控制项。
12. 随机遍历模式下，若当前预览文件因过滤条件变化被移出结果集，预览自动切换必须优先沿用随机队列的下一项，不得退化为顺序列表回退。
13. 全屏图片预览允许提供水平滑动作为“上一项/下一项”的触摸入口，但其导航结果必须与既有 `prev/next` 语义完全一致。

### 4.1 跨组件共享语义定义 (Shared Semantics)

1. 遍历模式
   - 值域：顺序模式 / 随机模式
   - 作用对象：当前媒体集合（图片/视频）
2. 预览播放状态
   - 状态：停止 / 播放中 / 因可见性暂停
3. 预览表现面
   - 形态：侧栏预览 / 全屏预览
4. 自动播放配置
   - 语义：自动播放开关 + 间隔秒数
5. 随机遍历状态
   - 语义：当前随机队列、已播放历史、当前位置游标

共享原则：

1. 侧栏预览与全屏预览必须共用同一套预览播放状态机。
2. 禁止为两种表现面维护独立且不一致的播放逻辑。

## 5. 功能需求 (FR)

1. `FR-PB-01` 系统必须支持自动播放启停，默认关闭。
2. `FR-PB-02` 自动播放间隔必须支持 `1-10` 秒，默认 `3` 秒。
3. `FR-PB-03` 图片必须按当前间隔切换到下一项。
4. `FR-PB-04` 视频必须在自然播放结束后切换到下一项。
5. `FR-PB-05` 播放到末尾时必须循环到首项继续播放。
6. `FR-PB-06` 系统必须支持顺序模式与随机模式两种遍历模式切换。
7. `FR-PB-07` 随机模式单轮内不得重复，轮末必须自动重洗牌。
8. `FR-PB-08` 手动上一项/下一项与自动播放下一项必须共用同一遍历策略。
9. `FR-PB-09` 预览快捷键 `P/T/Space/[ / ]/J/L/R/Esc` 在侧栏与全屏语义必须一致。
10. `FR-PB-10` 输入控件聚焦时，非全局预览快捷键必须失效。
11. `FR-PB-11` 页面失焦或不可见时必须暂停播放，恢复可见后按用户意图恢复。
12. `FR-PB-12` 媒体集合变化（筛选/目录切换）时必须重建随机队列并保持行为一致。
13. `FR-PB-13` 预览播放相关错误必须可见反馈，且不得阻断继续浏览。
14. `FR-PB-14` 关闭预览后不得因随机队列或播放副作用被强制重新打开。
15. `FR-PB-15` 视频快进步长必须支持 `3/5/10` 秒，默认 `5` 秒，并支持跨刷新持久化。
16. `FR-PB-16` 视频倍速必须支持 `0.5/1/3/5`，默认 `1`，并支持快捷键循环切换与跨刷新持久化。
17. `FR-PB-17` `J/L` 仅在视频预览生效，调整 `currentTime` 时必须做 `0..duration` 边界钳制。
18. `FR-PB-18` 随机遍历模式下当前预览项因过滤变化被移出可见集合时，若仍存在可预览媒体项，系统必须按随机队列续播下一项。
19. `FR-PB-19` 遍历模式必须跨刷新持久化；刷新后恢复最近一次用户选择的顺序/随机模式，且不得因恢复设置自动打开预览。
20. `FR-PB-20` 全屏图片预览若提供横向滑动切换，必须复用既有上一项/下一项、顺序/随机与边界处理语义，不得新增 swipe 专属规则。

## 6. 验收标准 (AC)

1. `AC-PB-01` 图片在自动播放开启后按设定间隔连续切换，改速后下一轮按新间隔生效。
2. `AC-PB-02` 视频项不会被图片计时器抢先切换，视频自然播放结束后正确切到下一项。
3. `AC-PB-03` 到达末项后可循环播放，顺序模式前后行为一致。
4. `AC-PB-04` 随机模式单轮无重复，轮末自动重建并继续播放。
5. `AC-PB-05` 随机模式下手动上一项按历史回退，下一项按随机队列前进。
6. `AC-PB-06` `P/T/Space/[ / ]/J/L/R/Esc` 在侧栏与全屏表现面语义一致。
7. `AC-PB-07` 输入框聚焦时快捷键保护生效，不触发预览控制动作。
8. `AC-PB-08` 页面失焦暂停、回前台恢复行为正确，且不改变用户启停意图。
9. `AC-PB-09` 预览关闭后不会被随机队列同步逻辑强制重开。
10. `AC-PB-10` 自动播放与手动切换过程中，网格选中跟随稳定，无明显闪烁回归。
11. `AC-PB-11` 视频预览中 `J/L` 按当前步长快退/快进，长按可连续触发且不越界。
12. `AC-PB-12` 视频预览中 `R` 按 `1 -> 3 -> 5 -> 0.5 -> 1` 循环，长按不会一次跳过多个倍率档位。
13. `AC-PB-13` 快进步长与倍速在顶部控件可见可调，且刷新后恢复上次值。
14. `AC-PB-14` 在随机模式 + 标注过滤（如“包含标签：未标注”）下，当前文件打标后移出结果集时，预览自动切换到随机队列中的下一项，而非顺序列表下一项。
15. `AC-PB-15` 用户切换遍历模式并刷新页面后，预览控件恢复为上次选择的顺序/随机状态；若刷新前预览关闭，刷新后不得仅因恢复遍历模式而自动打开预览。
16. `AC-PB-16` 全屏图片预览中的横滑切换与按钮/快捷键触发的上一项或下一项结果一致；视频预览不因该入口新增独立 swipe 语义。

## 7. 默认值与一致性约束 (Defaults & Consistency)

1. 默认自动播放间隔为 `3` 秒。
2. 默认自动播放状态为关闭。
3. 默认遍历模式为顺序模式。
4. 自动播放开关与自动播放间隔仅在当前会话有效，不做跨刷新持久化。
5. 遍历模式必须跨刷新持久化；当不存在有效持久化值时回退为默认顺序模式。
6. 视频快进步长与视频倍速必须跨刷新持久化，且侧栏与全屏共享同一配置。
7. 快捷键配置必须保持单一来源，文档说明必须与实际行为一致。
8. 侧栏与全屏仅允许表现层差异，不允许播放语义差异。

## 8. 失败与降级行为 (Failure & Degradation)

1. 文件读取失败、媒体解码失败、播放切换失败必须在预览区可见反馈。
2. 单项播放失败不得阻塞继续浏览或后续切换。
3. 网关能力离线不影响本规范定义的本地预览播放主链路。
4. 随机队列重建失败时，系统应回退到可继续浏览的安全状态，不得造成 UI 卡死或强制重开预览。

## 9. 测试与验收场景 (Test Scenarios)

1. 图片连续自动切换与变速生效验证。
2. 视频自然播放结束后的切换验证，且不被图片计时器抢切。
3. 末尾循环验证（顺序模式）。
4. 随机单轮不重复与轮末重洗牌验证。
5. 随机历史回退与队列前进验证。
6. 侧栏/全屏快捷键语义一致性验证（`P/T/Space/[ / ]/J/L/R/Esc`）。
7. 输入焦点保护验证（搜索框聚焦时不触发预览快捷键）。
8. 页面可见性切换下的暂停/恢复验证。
9. 预览关闭后随机副作用不重开验证。
10. 自动播放与手动切换下网格高亮稳定性回归验证。
11. 视频快进步长配置（`3/5/10`）与 `J/L` 边界钳制验证。
12. 视频倍速配置（`0.5/1/3/5`）与 `R` 循环顺序验证。
13. 刷新后快进步长与倍速持久化恢复验证。
14. 刷新后遍历模式持久化恢复验证，且不误打开原本关闭的预览。
15. 全屏图片预览中的横滑切换与按钮/快捷键 `prev/next` 结果一致性验证。
16. 视频全屏预览下左右拖动不会触发上一项/下一项导航验证。

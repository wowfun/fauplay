# 103 Grid Multi-selection 文件网格多选规范

## 1. 目的 (Purpose)

定义 Fauplay 文件网格区多选行为契约（Grid Multi-selection Contract），统一复选框多选、范围选择与快捷键语义，并约束与预览交互的兼容优先级。

## 2. 关键术语 (Terminology)

- 网格多选（Grid Multi-selection）
- 复选框选择（Checkbox Selection）
- 活跃项（Active Item）
- 勾选集合（Checked Set）
- 范围选择锚点（Range Anchor）
- 范围选择（Range Selection）
- 左键框选（Lasso Selection）

术语值映射（仅用于命名参考）：

1. 活跃项值映射：当前键盘焦点或最后一次主选择项。
2. 范围锚点值映射：最后一次非 `Shift` 主选择项路径。

## 3. 范围与非目标 (In Scope / Out of Scope)

范围内（Phase 1）：

1. 文件与目录卡片常驻复选框。
2. 勾选集合维护与状态栏展示。
3. `Ctrl/Cmd + A` 全选当前可见项（文件+目录）。
4. `Esc` 在无打开预览时清空勾选集合。
5. `Shift + 单击` 与 `Shift + 方向键` 范围选择。
6. 目录切换与筛选变更下的选择集合生命周期规则。

范围外（本轮不实现）：

1. 左键框选（Lasso Selection）仅作为 Phase 2 规划。
2. 持久化勾选集合到刷新后会话。
3. `Ctrl+Shift` 追加范围等高级组合语义。

## 4. 用户可见行为契约 (User-visible Contract)

1. 网格中每个文件与目录卡片均显示复选框，用户可独立勾选/取消。
2. 普通单击卡片保持现有行为：
   - 目录：进入目录。
   - 文件：打开侧栏预览。
3. `Ctrl/Cmd + 单击` 卡片主体仅切换勾选，不触发目录进入或预览打开。
4. `Shift + 单击` 卡片主体执行范围选择（覆盖当前勾选集合），不触发目录进入或预览打开。
5. `Shift + 方向键` 扩展范围选择，松开 `Shift` 后再同步预览到最终活跃文件。
6. `Ctrl/Cmd + A` 全选当前可见网格项（文件+目录）。
7. `Esc` 在无打开预览时清空勾选集合；有打开预览时优先执行预览关闭。
8. 切换目录后清空勾选集合；筛选变化后仅保留仍可见的勾选项交集。

## 5. 功能需求 (FR)

1. `FR-MS-01` 系统必须为文件与目录卡片提供常驻复选框入口。
2. `FR-MS-02` 系统必须支持多选勾选集合维护，并向状态栏暴露“已选数量”。
3. `FR-MS-03` 系统必须支持 `Ctrl/Cmd + A` 全选当前可见项。
4. `FR-MS-04` 系统必须支持在“无打开预览”时 `Esc` 清空勾选集合。
5. `FR-MS-05` 系统必须支持 `Shift + 单击` 范围选择，语义为“覆盖当前勾选集合”。
6. `FR-MS-06` 系统必须支持 `Shift + 方向键` 范围选择，且预览延迟到 `Shift` 松开时同步。
7. `FR-MS-07` 范围锚点必须使用最后一次非 `Shift` 主选择项。
8. `FR-MS-08` 切换目录时必须清空勾选集合与范围锚点。
9. `FR-MS-09` 筛选变化时必须将勾选集合收敛到当前可见项交集。
10. `FR-MS-10` 网格多选能力不得破坏既有目录进入、预览打开与全屏打开行为。

## 6. 验收标准 (AC)

1. `AC-MS-01` 普通单击目录仍进入目录，普通单击文件仍打开侧栏预览。
2. `AC-MS-02` 复选框勾选/取消不触发目录进入或预览打开。
3. `AC-MS-03` `Ctrl/Cmd + 单击` 仅切换勾选，不触发原单击行为。
4. `AC-MS-04` `Shift + 单击` 在同一网格内正确覆盖范围勾选。
5. `AC-MS-05` `Shift + 方向键` 可连续扩展范围，松开 `Shift` 后才切换预览。
6. `AC-MS-06` `Ctrl/Cmd + A` 全选当前可见文件与目录。
7. `AC-MS-07` 切换目录后勾选集合清空。
8. `AC-MS-08` 筛选后勾选集合只保留仍可见项。
9. `AC-MS-09` 有预览打开时按 `Esc` 先关闭预览，不清空勾选。
10. `AC-MS-10` 无预览打开时按 `Esc` 清空勾选集合。

## 7. Phase 2 规划：左键框选 (Lasso Selection)

1. 左键框选在 Web 端可行，但本轮不实现。
2. 实现前置约束：
   - 基于 Pointer Events 处理拖拽框选。
   - 兼容虚拟列表可见区命中与自动滚动。
   - 保持与复选框/范围选择一致的勾选集合语义。
3. 任何 lasso 接入不得改变本规范 Phase 1 的快捷键与优先级契约。

## 8. 默认值与一致性约束 (Defaults & Consistency)

1. 复选框默认常驻显示。
2. 范围选择默认覆盖当前勾选集合。
3. `Esc` 优先级保持与 `003-ui-ux` 一致：先预览关闭，再网格清选。
4. 快捷键配置与文档必须保持单一来源一致性。

# 003 Thumbnail - 网格缩略图规范

## 1. 背景与目标

当前网格缩略图能力已可用，但在大目录与快速滚动场景下，仍需要统一的行为契约与验收口径。  
本规范用于定义稳定的产品与策略要求，确保缩略图体验在后续迭代中保持一致、可验证、可回归。

## 2. 范围与边界

### 2.1 范围内

- 文件网格区中的图片/视频缩略图行为。
- 缩略图触发策略、加载优先级、失败降级与缓存一致性。
- 缩略图体验在普通目录视图与平铺视图中的一致性要求。
- 缩略图尺寸档位切换能力：`默认值（程序决定）`、`256x256`、`512x512`。

### 2.2 范围外（非目标）

- 预览窗格与全屏预览的媒体加载策略。
- 后端服务或本地网关改造。
- 代码级实现方案、具体函数与文件级实现步骤。

## 3. 用户可见行为契约

- 文件卡片默认显示占位态。
- 缩略图进入加载流程后显示“加载中”反馈。
- 加载成功后显示缩略图，失败时回退到可识别的文件类型占位图。
- 快速滚动时，当前可见区域的体验优先于不可见区域。
- 页面刷新后允许重新生成缩略图，不要求跨刷新保留。
- 用户可切换缩略图尺寸档位：`默认值（程序决定）`、`256x256`、`512x512`。
- 切换尺寸后应保持网格可操作，避免明显卡顿或整页阻塞。

## 4. 功能需求（FR）

- `FR-THUMB-01` 系统必须在“接近可见或进入可见”的时机触发缩略图加载，而非无差别全量触发。
- `FR-THUMB-02` 系统必须保证可见区域项的缩略图加载优先级高于非可见区域项。
- `FR-THUMB-03` 系统必须限制同时进行的缩略图生成任务数量，避免影响网格交互流畅性。
- `FR-THUMB-04` 系统必须对重复请求进行去重，避免同一对象被重复生成。
- `FR-THUMB-05` 系统必须定义缓存有效性判定规则，确保文件变化后可获得更新后的缩略图。
- `FR-THUMB-06` 单项缩略图失败不得阻塞其他文件项的渲染与交互。
- `FR-THUMB-07` 缩略图策略必须保持纯 Web 约束，不引入后端强依赖。
- `FR-THUMB-08` 系统必须提供三种缩略图尺寸档位：`默认值（程序决定）`、`256x256`、`512x512`。
- `FR-THUMB-09` 当用户切换尺寸档位时，系统必须保证网格交互可持续响应。
- `FR-THUMB-10` 切换到更高尺寸档位时，系统必须支持渐进更新，不要求全量同步完成后再渲染。
- `FR-THUMB-11` 不同尺寸档位的缩略图结果必须互不混淆，避免错误复用导致的显示失真或陈旧。

## 5. 验收标准（AC）

### 5.1 触发与优先级

- `AC-THUMB-TRI-01` 非可见区域项在未接近可见区前不应抢占可见区资源。
- `AC-THUMB-TRI-02` 可见区域项应在可接受时间内完成加载或进入降级态。

### 5.2 性能与稳定性

- `AC-THUMB-PERF-01` 大目录滚动过程中，网格滚动与选中行为无明显卡顿。
- `AC-THUMB-PERF-02` 连续滚动时不出现由缩略图任务引发的整体渲染抖动。

### 5.3 缓存一致性

- `AC-THUMB-CACHE-01` 同一文件在未变化时可复用缓存结果。
- `AC-THUMB-CACHE-02` 文件发生变化后，不应长期展示陈旧缩略图。

### 5.4 失败降级

- `AC-THUMB-ERR-01` 单项缩略图失败时，系统应展示可识别降级态并保持可操作。
- `AC-THUMB-ERR-02` 图片或视频单项失败不得影响其他项继续加载。

### 5.5 尺寸档位

- `AC-THUMB-SIZE-01` 系统提供且仅提供三档：`默认值（程序决定）`、`256x256`、`512x512`。
- `AC-THUMB-SIZE-02` 用户切换尺寸后，网格仍可滚动、选择与打开，不出现明显卡顿。
- `AC-THUMB-SIZE-03` 切换到 `512x512` 时，允许分批更新，不要求一次性完成所有卡片更新。
- `AC-THUMB-SIZE-04` 不同尺寸档位切换后，显示结果与当前档位一致，不出现跨档位误用。

## 6. 非功能约束

- 纯 Web 约束：不依赖后端服务才能完成核心缩略图体验。
- 可回退：缩略图策略升级需支持分阶段替换与回退。
- 可观测：关键失败与性能退化场景可被定位和排查。
- 可测试：需求项应可映射到自动化或可重复手工验收场景。
- 默认值语义：`默认值（程序决定）` 为策略位，不固定像素值，可随性能策略演进。

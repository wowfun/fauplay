# 003 缩略图待办与实现计划

- [ ] 建立缩略图规范与任务清单双文档结构（`spec.md` + `todo.md`）。

## 2. 待办（Todo）

- [ ] 尺寸档位能力：支持 `默认值（程序决定）`、`256x256`、`512x512` 三档切换。
- [ ] 尺寸切换体验：切档过程保持网格可交互，避免明显卡顿。
- [ ] 尺寸一致性：确保不同尺寸档位的缩略图结果互不混淆。
- [ ] 触发机制改造：从“卡片挂载触发”升级为“接近可见/进入可见触发”。
- [ ] 优先级策略：明确可见区任务优先于预取任务。
- [ ] 并发控制：为缩略图生成引入可配置并发上限与任务队列。
- [ ] 去重策略：同一缓存键请求合并，避免重复生成。
- [ ] 缓存键策略：从“仅路径”升级为“路径 + 文件变化维度”。
- [ ] 失效策略：定义文件变更、目录切换、异常中断下的缓存失效规则。
- [ ] 降级策略：单项失败时统一回退到占位态并可重试。
- [ ] 稳定性验证：在大目录和快速滚动场景做性能回归。
- [ ] 文档同步：补充 README 与 troubleshooting 的缩略图排障入口。

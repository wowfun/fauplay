# 003 触摸交互细则 (Touch Interactions Reference)

## 目的

定义输入模式主轴下的具体触摸手势细则，作为 [`input-modes.md`](./input-modes.md) 的下挂参考文档；本文件只补充触摸入口，不扩展新的浏览、播放或导航语义。

## 范围与边界

范围内：

1. 全屏图片预览中的水平滑动切换。
2. 触摸手势与轻点、短拖、纵向滚动之间的冲突规避。
3. 手势与既有 `prev/next` 语义之间的对齐关系。

范围外：

1. 宽/窄视口布局规则（归属 [`viewport-modes.md`](./viewport-modes.md)）。
2. `keyboard | touch | hybrid` 三态输入模式本身（归属 [`input-modes.md`](./input-modes.md)）。
3. 视频播放器手势体系、全局返回手势、浏览器系统手势与 OS 级手势兼容策略。
4. pinch zoom、双指操作、长按菜单等更丰富的触摸语义。

## 用户可见行为契约

1. 全屏图片预览必须允许通过水平滑动在当前预览序列中切换上一项/下一项。
2. 语义固定为：左滑 = 下一项，右滑 = 上一项。
3. 该手势只是在全屏图片预览中提供 `prev/next` 的触摸入口，不引入 swipe 专属遍历规则。
4. 视频全屏预览在 v1 不启用横滑切换，避免与播放器原生手势、拖动与浏览器媒体控件冲突。

## 触发与冲突规避规则

1. 横滑切换只允许在全屏图片预览的主体视口区域生效。
2. 工具栏、关闭按钮、控制条等明确操作区上的拖动不得触发导航。
3. 只有当横向位移明显大于纵向位移，且超过实现定义的最小阈值时，才允许认定为切换手势。
4. 轻点、短拖、未达阈值的横向滑动不得触发切换。
5. 纵向滚动意图必须优先于横向导航，避免在阅读长图或页面滚动场景中误切换。

## 语义对齐约束

1. 横滑切换必须复用 [`../100-preview-playback/spec.md`](../100-preview-playback/spec.md) 中既有的上一项/下一项、顺序/随机与边界行为。
2. 手势触发后的结果必须与点击“上一项/下一项”按钮或对应快捷键一致。
3. 触摸手势不得绕开现有预览状态机，不得维护独立的 swipe 序列。

## 功能需求 (FR)

1. `FR-TI-01` 系统必须在全屏图片预览中支持水平滑动切换上一项/下一项。
2. `FR-TI-02` 左滑必须映射为“下一项”，右滑必须映射为“上一项”。
3. `FR-TI-03` 横滑切换必须只在全屏图片预览主体视口内生效。
4. `FR-TI-04` 视频全屏预览在 v1 不得启用横滑切换。
5. `FR-TI-05` 只有在横向位移明显大于纵向位移且达到阈值后，系统才允许触发导航。
6. `FR-TI-06` 轻点、短拖、纵向滑动与工具栏区域拖动不得误触发导航。
7. `FR-TI-07` 横滑切换必须复用既有 `prev/next` 语义，不得新增独立 swipe 播放规则。

## 验收标准 (AC)

1. `AC-TI-01` 在全屏图片预览中左滑会切到下一项，右滑会切到上一项。
2. `AC-TI-02` 轻点、短距离拖动与不满足阈值的手势不会触发切换。
3. `AC-TI-03` 纵向滑动不会被误判为左右导航。
4. `AC-TI-04` 在工具栏或控制区拖动时不会触发图片切换。
5. `AC-TI-05` 视频全屏预览下左右拖动不会触发上一项/下一项导航。
6. `AC-TI-06` 横滑后的目标项、随机/顺序行为与现有 `prev/next` 逻辑完全一致。

# 003 输入模式细则 (Input Modes Reference)

## 目的

定义工作区在不同输入能力下的交互入口与守卫规则，作为 [`spec.md`](./spec.md) 的主轴文档之一；本文件只讨论键盘、触控与兼具输入，不讨论宽/窄视口或完全访问/远程只读差异。

## 范围与边界

范围内：

1. `keyboard`、`touch`、`hybrid` 三种输入模式的定义。
2. 快捷键、hover、右键与精细拖拽在不同输入模式下的可依赖程度。
3. 多选、预览关闭、上一项/下一项与批量操作在不同输入模式下的入口要求。

范围外：

1. 宽/窄视口布局转译（归属 [`viewport-modes.md`](./viewport-modes.md)）。
2. `full-access` 与 `remote-readonly` 的能力显隐差异（归属 [`access-modes.md`](./access-modes.md)）。
3. 具体触摸手势细节（归属 [`touch-interactions.md`](./touch-interactions.md)）。

## 模式定义

1. `keyboard`
   - 可以依赖硬件键盘、hover、右键与精细指针
   - 是默认基线输入模式
2. `touch`
   - 不得依赖硬件键盘、hover、右键与精细指针
   - 核心路径必须通过显式触摸 UI 闭环完成
3. `hybrid`
   - 键盘与触控两套入口同时成立
   - 触摸入口与快捷键入口不得互相遮蔽或冲突

## 用户可见行为契约

1. 输入模式是交互入口差异，不是新的信息架构。
2. `keyboard` 下可继续使用快捷键、hover reveal 与精细拖拽作为高频入口。
3. `touch` 下快捷键只能视为加速器而非唯一入口；所有核心路径都必须有稳定可点击入口。
4. `touch` 下 hover-only 能力必须提供显式 tap 等价入口。
5. `touch` 下多选必须允许通过显式“选择模式”或同等清晰的触摸批量选择模式进入。
6. `touch` 下预览关闭、上一项、下一项与返回工作区都必须具备可点控件；触摸手势只作为增强入口。
7. `hybrid` 下键盘与触摸入口都必须成立；系统不得因为存在键盘而撤掉触摸闭环，也不得因为存在触摸而破坏快捷键一致性。
8. 输入模式判定必须基于运行时能力（如 coarse/fine pointer、hover、触控存在性）而不是 UA 设备猜测。

## 交互补位原则

1. 核心动作显式可点：
   - 至少覆盖关闭预览、返回工作区、上一项、下一项、切换视图与打开筛选
2. Hover 语义有触摸等价入口：
   - hover reveal 的卡片动作、更多菜单、帮助提示与次级入口必须可通过触摸显式发现
3. 多选有显式模式：
   - 无法依赖 `Ctrl/Cmd` 与 `Shift` 时，勾选入口与批量操作入口必须稳定可见
4. 精细拖拽有受控降级：
   - resize handle、窄把手与小热区不能成为关键交互的唯一方式

## 功能需求 (FR)

1. `FR-IM-01` 系统必须固定支持 `keyboard | touch | hybrid` 三种输入模式。
2. `FR-IM-02` `keyboard` 必须作为默认基线输入模式。
3. `FR-IM-03` `touch` 下核心操作不得仅通过快捷键暴露。
4. `FR-IM-04` `touch` 下 hover-only 控件必须提供可点击或可轻触发现的等价入口。
5. `FR-IM-05` `touch` 下必须提供显式多选模式或同等清晰的触摸批量选择模式。
6. `FR-IM-06` `touch` 下预览关闭、上一项、下一项与返回工作区不得依赖快捷键作为唯一入口。
7. `FR-IM-07` `hybrid` 下快捷键、触摸按钮与触摸手势可并存，但不得产生冲突或互斥。
8. `FR-IM-08` 具体触摸手势语义统一归属 [`touch-interactions.md`](./touch-interactions.md)，输入模式主文档不得再复制同一套手势规则。
9. `FR-IM-09` 运行时必须提供统一输入模式判定入口，产出 `keyboard | touch | hybrid` 三态，供工作区表现层复用。

## 验收标准 (AC)

1. `AC-IM-01` 在 `keyboard` 输入模式下，既有快捷键、hover 与精细拖拽入口保持可用。
2. `AC-IM-02` 在 `touch` 输入模式下，用户仍可完成目录浏览、点击文件、进入预览、关闭预览与返回工作区。
3. `AC-IM-03` 在 `touch` 输入模式下，关闭预览、上一项与下一项均有稳定可点入口，不依赖快捷键。
4. `AC-IM-04` 在 `touch` 输入模式下，依赖 hover 才出现的卡片动作或菜单可以被显式发现并触发。
5. `AC-IM-05` 在 `touch` 输入模式下，用户可进入显式多选模式，并通过触摸完成勾选与批量操作。
6. `AC-IM-06` 在 `hybrid` 输入模式下，快捷键链路与触摸链路都可用，且不会相互遮蔽。
7. `AC-IM-07` 输入模式由能力判定得出；桌面浏览器窄窗口、平板与手机浏览器不会因 UA 分支得到不同模式真源。

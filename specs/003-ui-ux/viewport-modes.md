# 003 视口模式细则 (Viewport Modes Reference)

## 目的

定义工作区在不同视口宽度下的表现态约束，作为 [`spec.md`](./spec.md) 的主轴文档之一；本文件只讨论宽/窄视口差异，不讨论输入能力或访问权限。

## 范围与边界

范围内：

1. `wide` / `narrow` 两种视口模式的定义。
2. 窄屏触发条件与覆盖对象。
3. 主视图、预览入口与次级面板在宽/窄视口下的转译原则。

范围外：

1. 键盘、触控或兼具输入下的入口差异（归属 [`input-modes.md`](./input-modes.md)）。
2. `full-access` 与 `remote-readonly` 的能力显隐差异（归属 [`access-modes.md`](./access-modes.md)）。
3. 具体触摸手势语义（归属 [`touch-interactions.md`](./touch-interactions.md)）。

## 模式定义

1. `wide`
   - 触发条件：`viewport width > 768px`
   - 语义：默认宽视口工作区表现态
2. `narrow`
   - 触发条件：`viewport width <= 768px`
   - 语义：窄视口工作区表现态
3. 视口模式只按当前视口宽度判断，不按设备类型、UA 或“是否手机/平板/桌面端”判断。

## 用户可见行为契约

1. `wide + keyboard + full-access` 是工作区默认基线表现态。
2. `narrow` 不是独立业务域，而是当前工作区在窄视口下的表现态。
3. `wide` 下默认允许保持目录主区、预览面板区与底部结果面板区的桌面式并存。
4. `narrow` 下主视图默认聚焦文件网格与媒体浏览主路径，不再依赖桌面常驻侧栏作为主入口。
5. `narrow` 下点击图片或视频时，应允许直接进入全屏预览。
6. `narrow` 下收藏、历史、筛选、人物与帮助等次级能力可转为抽屉、overlay 或 bottom sheet。
7. `narrow` 下若人物等次级能力在 `wide` 中依赖列表/详情并列布局，必须允许转为单列 staged flow：先进入列表，再进入详情，或通过显式返回动作回到列表；不得继续把桌面 split-pane 硬塞进窄视口。
8. 视口模式只能改变布局和主路径入口，不得擅自改写预览遍历、播放与工具语义。
9. 运行时实现应优先按视口主壳拆分：`wide` 承接桌面式并列工作区，`narrow` 承接紧凑工作区；输入与访问差异再通过表现层 profile 注入。

## 区域转译原则

1. A 顶部工具区在 `wide` 与 `narrow` 下都保留为全局入口带，但 `narrow` 下应优先保留目录语境、返回与必要状态入口。
2. B1 文件网格区在两种模式下都保持为工作区主承载区；`narrow` 下其优先级更高。
3. B2 预览面板区在 `wide` 下可常驻；在 `narrow` 下可转为“点击媒体后进入 D 全屏预览区”的主路径入口。
4. B3 底部结果面板区在 `wide` 下保持既有桌面表现；在 `narrow` 下可转为覆盖式结果面板或临时抽屉，但不得改变其“工作区结果投射文件视图”的语义。
5. 人物面板在 `wide` 下可保留列表/详情并列布局；在 `narrow` 下应优先转为全宽 overlay，并允许通过 list/detail staged flow、显式返回按钮与更紧凑的人脸网格完成闭环。
6. C 底部状态区若在 `narrow` 实现中做可见性裁剪，必须显式记录为专题 Delta，不得隐式删除。

## 功能需求 (FR)

1. `FR-VM-01` 系统必须固定支持 `wide | narrow` 两种视口模式。
2. `FR-VM-02` `narrow` 的默认触发阈值必须为 `viewport width <= 768px`。
3. `FR-VM-03` 视口模式判定不得依赖设备类型或 UA 猜测。
4. `FR-VM-04` `wide` 必须作为默认基线表现态。
5. `FR-VM-05` `narrow` 下图片与视频点击必须可直接进入全屏预览。
6. `FR-VM-06` `narrow` 下次级能力必须允许通过抽屉、overlay 或 bottom sheet 进入，而不是要求常驻侧栏。
7. `FR-VM-07` 视口模式差异不得改变工作区结果投射、预览遍历与插件作用域的核心语义。
8. `FR-VM-08` `narrow` 下的次级 overlay 必须允许将桌面并列布局转译为单列 staged flow 或等价的全宽表现态。
9. `FR-VM-09` 工作区运行时必须允许按 `wide | narrow` 拆出独立主壳，而不是把两种布局长期混写在同一个巨大组件中。

## 验收标准 (AC)

1. `AC-VM-01` 当视口宽度从 `> 768px` 缩小到 `<= 768px` 时，工作区切换到 `narrow` 规则；反向放宽后恢复 `wide` 规则。
2. `AC-VM-02` 手机浏览器、桌面窄窗口与小平板竖屏命中同一套 `narrow` 规则，不存在设备特判分叉。
3. `AC-VM-03` `wide` 下既有桌面式布局保持原样，不因视口专题发生语义回归。
4. `AC-VM-04` `narrow` 下点击图片或视频时，用户可直接进入全屏预览而不依赖常驻侧栏预览。
5. `AC-VM-05` `narrow` 下收藏、历史、筛选、人物等次级能力可通过临时面板进入，且不会长期挤占文件网格主视口。
6. `AC-VM-06` `narrow` 下的人物面板不会继续保留桌面式列表/详情并列布局，而是以全宽 overlay 和显式 list/detail 切换完成闭环。
7. `AC-VM-07` `wide` 与 `narrow` 的主布局差异可由不同主壳承接，但二者继续共享同一套工作区状态与预览状态机。

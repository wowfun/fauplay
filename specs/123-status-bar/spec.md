# 123 Status Bar 状态栏规范

## 1. 目的 (Purpose)

定义 Fauplay 底部状态栏（Status Bar）契约，统一工作区底部状态区在目录浏览、结果投射与全屏预览下的信息结构、父目录路径展示与可见性约束，确保：

1. 状态栏持续提供当前活动表面的可见项与已选项统计。
2. 单文件元信息展示固定收敛为 `大小 / 修改时间 / 路径（父目录）`。
3. 全屏预览时继续复用同一底部状态区，而不是创建第二套独立状态栏。

## 2. 范围与非目标 (In Scope / Out of Scope)

范围内：

1. 状态栏统计与单文件元信息展示契约。
2. 父目录路径解析优先级与回退规则。
3. 目录表面、结果投射表面与全屏预览下的状态栏目标文件语义。
4. 全屏预览对底部状态区的遮挡约束。

范围外：

1. 新增状态栏交互按钮、菜单或快捷键。
2. 复制路径、右键菜单或状态栏点击行为。
3. 重新设计整个工作区视觉风格。

## 3. 用户可见行为契约 (User-visible Contract)

1. 底部状态栏始终展示当前活动表面的 `可见` 与 `已选` 计数。
2. 当存在明确目标文件时，状态栏必须额外展示：
   - `大小`
   - `修改`
   - `路径`（显示该文件父目录）
3. 当侧栏预览或全屏预览存在当前预览文件时，状态栏单文件元信息目标必须优先跟随该预览文件，不要求该文件处于勾选状态。
4. 当不存在当前预览文件时，状态栏可回退为当前活动表面中“单个勾选文件”的元信息展示；统计口径始终保持当前活动表面不变。
5. 路径文本默认单行展示，超出宽度时必须截断，并通过 `title` 或等效方式暴露完整值。
6. 多选时保留 `可见 / 已选` 统计，但普通浏览态不显示单文件元信息。

## 4. 父目录路径解析契约 (Parent Path Resolution Contract)

1. 父目录路径解析优先级固定为：
   - `displayPath`（仅当其本身是绝对路径）
   - `absolutePath`
   - `getBoundRootPath(rootId) + relativePath`
   - root 相对父目录
   - `当前根目录`
2. `displayPath` 若为相对路径，不得抢占绝对路径来源。
3. 对当前 root 下的相对路径文件，若已绑定 root 绝对路径，状态栏必须拼出绝对父目录。
4. 当文件位于 root 顶层且无法解析出绝对父目录时，状态栏固定显示 `当前根目录`。
5. 结果投射、回收站或跨 Root 文件若携带 `absolutePath`，状态栏必须优先使用其父目录，而不是误用当前 root 相对路径。

## 5. 全屏一致性契约 (Fullscreen Consistency Contract)

1. 全屏预览必须继续复用底部状态区（C），不得新增独立的“全屏专用状态栏”。
2. 全屏覆盖层必须为底部状态区预留固定高度，不得遮挡状态栏。
3. 全屏加载态、错误态、不可预览态与媒体/文本就绪态均必须满足上述可见性约束。
4. 状态栏高度必须由共享常量统一驱动，避免预览覆盖层与状态栏实际高度分叉。

## 6. 功能需求 (FR)

1. `FR-SB-01` 系统必须在状态栏展示当前活动表面的 `可见` 与 `已选` 计数。
2. `FR-SB-02` 系统必须支持状态栏展示单文件 `大小 / 修改时间 / 父目录路径` 元信息。
3. `FR-SB-03` 当存在当前预览文件时，状态栏单文件元信息必须优先跟随该预览文件，不要求勾选。
4. `FR-SB-04` 当不存在当前预览文件时，状态栏可回退展示单个勾选文件元信息。
5. `FR-SB-05` 父目录路径解析必须遵循固定优先级，并在可用时优先展示绝对路径。
6. `FR-SB-06` 当路径文本过长时，状态栏必须截断显示但保留完整值提示。
7. `FR-SB-07` 全屏预览不得遮挡底部状态区。

## 7. 验收标准 (AC)

1. `AC-SB-01` 打开侧栏预览后，即使当前预览文件未被勾选，状态栏仍显示该文件的大小、修改时间与父目录路径。
2. `AC-SB-02` 文件位于当前 root 顶层且 root 已绑定绝对路径时，状态栏显示该 root 的绝对父目录；未绑定时显示 `当前根目录`。
3. `AC-SB-03` 在结果投射标签中选中文件时，若该文件提供 `absolutePath` 或绝对 `displayPath`，状态栏显示对应绝对父目录而不是当前 root 相对路径。
4. `AC-SB-04` 多选时若存在当前预览文件，状态栏继续显示当前预览文件元信息；若不存在当前预览文件，则仅保留 `可见 / 已选` 统计。
5. `AC-SB-05` 打开全屏预览后，底部状态栏仍可见，且单文件元信息切换到当前全屏预览文件。
6. `AC-SB-06` 全屏的 loading / error / unsupported / text / image / video 各分支下，状态栏均不被遮挡。
7. `AC-SB-07` 当父目录路径过长时，状态栏主文案截断，但 hover 可看到完整路径。

## 8. 默认值与一致性约束 (Defaults & Consistency)

1. 本专题不新增快捷键，`src/config/shortcuts.ts` 与 `docs/shortcuts.md` 不变。
2. 状态栏继续保持只读展示区语义，不承载目录切换、预览控制或工具动作入口。
3. `大小` 字段必须正确区分 `0 B` 与未知大小，不得把 `0` 误显示为 `未知`。
